# 日志服务模块测试用例 (M07)

## 测试范围

日志服务模块负责应用日志的记录、存储、查询和导出，采用**SQLite + JSONL 归档**的分级存储策略。

## 发现的问题

1. **getCallerModule 实现脆弱** - 通过堆栈解析模块名，路径变化会导致解析失败
2. **导出全量日志风险** - `export()` 从 SQLite+JSONL 读取所有日志，大数据量可能 OOM
3. **归档频率过高** - 每 5 分钟检查归档，但归档是 IO 密集型操作，建议降低频率或改为空闲时执行

## 建议

1. 使用 `new Error().stack` 解析调用者时添加容错处理，解析失败时使用默认模块名
2. export() 添加流式写入支持，避免一次性加载所有日志到内存
3. 归档检查频率降低为每小时一次，或在系统空闲时执行
