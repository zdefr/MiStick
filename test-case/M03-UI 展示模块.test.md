# UI 展示模块测试用例 (M03)

## 测试范围

UI 展示模块负责便利贴窗口、扫码登录入口、二维码弹窗、设备列表渲染、设置面板和窗口管理。

**版本范围：** v0.1 首期基线

## 发现的问题

1. 登录弹窗需要同时覆盖 `pending / expired / failed / success` 四种状态，否则真实体验会断层。
2. “不可控设备”和“云控失败可回退设备”在 UI 上必须有不同的反馈。

## 建议

1. 把扫码登录的轮询状态机纳入组件快照和交互测试。
2. E2E 优先覆盖“扫码登录→同步→控制”主路径。
