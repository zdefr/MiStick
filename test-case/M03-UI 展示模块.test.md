# UI 展示模块测试用例 (M03)

## 测试范围

UI 展示模块负责便利贴窗口、登录入口、设备列表渲染、设置面板和窗口管理。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```
UI 展示模块测试用例
├── 组件测试
│   ├── ✓ StickyWindow 渲染与拖拽
│   ├── ✓ SessionBanner 展示登录态
│   ├── ✓ DeviceCard 展示可控/不可控状态
│   ├── ✓ SettingsModal 保存窗口与主题设置
│   └── ✓ LoginModal 提交登录表单
├── 状态管理测试
│   ├── ✓ sessionStore 登录/登出
│   ├── ✓ deviceStore 同步设备与错误状态
│   └── ✓ configStore 保存设置
├── 窗口管理测试
│   ├── ✓ 置顶切换
│   ├── ✓ 托盘最小化
│   └── ✓ 透明度实时更新
└── E2E 测试
    ├── ✓ 首次启动未登录
    ├── ✓ 登录后同步并渲染设备
    └── ✓ 控制成功后刷新状态
```

## 发现的问题

1. 对“不可控设备”的视觉反馈必须明确，否则用户会误以为控制失败。
2. 首屏是否显示缓存数据还是加载态，需要在实现时统一体验策略。

## 建议

1. 把“未登录态”“同步失败态”“不可控态”都纳入组件快照和交互测试。
2. E2E 优先覆盖首期主路径，不扩展到后续自动化能力。
