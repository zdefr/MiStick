# 设备管理模块测试用例 (M01)

## 测试范围

设备管理模块负责米家扫码登录、会话复用、云端同步设备与房间信息、本地缓存维护，以及向 UI 和 M02 提供统一设备元数据。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```text
设备管理模块测试用例
├── 扫码登录测试
│   ├── ✓ 启动二维码登录成功
│   ├── ✓ 二维码过期后可刷新
│   ├── ✓ 扫码成功后返回 SessionInfo
│   └── ✓ 认证文件存在时复用会话
├── 云端同步测试
│   ├── ✓ 有效会话下同步设备和房间
│   ├── ✓ 会话失效时返回 SESSION_EXPIRED
│   ├── ✓ 同步失败时保留旧缓存
│   └── ✓ force=true 时忽略缓存年龄
├── 缓存与映射测试
│   ├── ✓ 启动先加载缓存
│   ├── ✓ 设备映射出 controlCapability
│   ├── ✓ 设备映射出 cloud controlContext
│   └── ✓ 缺少本地上下文不影响同步成功
└── IPC 测试
    ├── ✓ auth:startQrLogin
    ├── ✓ auth:pollQrLogin
    ├── ✓ device:syncFromCloud
    └── ✓ device:getAll / device:getRooms
```

## 详细测试用例

### 扫码登录与会话测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|
| M01-001 | 启动二维码登录成功 | MiHome Bridge 可用 | 调用 `startQrLogin(region)` | 返回 `ticketId` 与 `qrCodeData` | P0 |
| M01-002 | 二维码登录成功 | 用户完成扫码 | 轮询 `pollQrLogin(ticketId)` | 返回 `status=success` 与 `SessionInfo` | P0 |
| M01-003 | 二维码过期 | 二维码超时 | 轮询 `pollQrLogin(ticketId)` | 返回 `QR_CODE_EXPIRED` 或 `status=expired` | P0 |
| M01-004 | 认证文件可复用会话 | `authStoragePath` 存在 | 应用重启后执行同步 | 无需重新扫码即可同步 | P0 |
| M01-005 | 会话缺失时禁止同步 | 无认证文件 | 调用 `syncFromCloud()` | 返回 `SESSION_MISSING` | P0 |

### 云端同步测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|
| M01-006 | 同步设备和房间成功 | 有效会话 | 调用 `syncFromCloud()` | 返回 `devices` 与 `rooms` | P0 |
| M01-007 | 同步成功后更新 SQLite 缓存 | 有效会话 | 调用 `syncFromCloud()` 后查询 DB | 缓存被覆盖更新 | P0 |
| M01-008 | 同步失败时保留旧缓存 | DB 中有旧缓存，桥接层报错 | 调用 `syncFromCloud()` | 返回错误，旧缓存仍可读取 | P0 |
| M01-009 | 强制同步忽略缓存年龄 | 缓存较新 | 调用 `syncFromCloud(true)` | 发起真实云端请求 | P1 |
| M01-010 | 自动同步按缓存年龄判断 | 有效会话 | 应用启动时执行 `loadFromCache()` | 超过阈值触发后台同步 | P0 |

### 数据映射与缓存测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|
| M01-011 | 云端设备映射为内部 Device | 云端返回设备数据 | 执行映射函数 | 生成统一 Device 结构 | P0 |

| M01-013 | 缺少本地上下文时仍同步成功 | 无本地 IP/Token | 执行同步 | `controlCapability.local=false` 且同步成功 | P0 |

## 发现的问题

1. 首期“同步成功”和“本地控制能力完整”不是一回事，测试需要接受部分设备只有云控或仅展示。
2. 二维码登录流程需要模拟轮询与超时，不能只测一次性成功路径。

## 建议

1. 先用 mock MiHome Bridge 稳定扫码登录与同步测试，再引入真实联调。
2. 将“会话复用成功”纳入 M01 的核心回归集。
