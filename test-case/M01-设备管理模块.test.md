# 设备管理模块测试用例 (M01)

## 测试范围

设备管理模块负责米家账号登录、云端同步设备与房间信息、本地缓存维护，以及向 UI 提供设备元数据。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```
设备管理模块测试用例
├── 登录测试
│   ├── ✓ 账号密码登录成功
│   ├── ✓ 错误密码返回认证失败
│   └── ✓ 登录成功后保存加密 Token
├── 云端同步测试
│   ├── ✓ 有效会话下同步设备和房间
│   ├── ✓ 会话失效时返回 SESSION_EXPIRED
│   ├── ✓ 同步失败时保留旧缓存
│   └── ✓ force=true 时忽略缓存年龄
├── 缓存测试
│   ├── ✓ 启动先加载缓存
│   ├── ✓ 缓存较新时可先展示旧数据
│   └── ✓ 同步成功后覆盖 SQLite 缓存
├── 数据映射测试
│   ├── ✓ 云端设备映射到内部 Device
│   ├── ✓ 未知 model 映射为 Unknown
│   └── ✓ 缺少控制上下文时 isControllable=false
└── IPC 测试
    ├── ✓ auth:login
    ├── ✓ device:syncFromCloud
    ├── ✓ device:getAll
    └── ✓ device:getRooms
```

## 详细测试用例

### 登录与会话测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|
| M01-001 | 账号密码登录成功 | 有效账号 | 调用 `login(credentials)` | 返回 `SessionInfo` 并写入配置 | P0 |
| M01-002 | 错误密码登录失败 | 密码错误 | 调用 `login(credentials)` | 返回认证失败错误 | P0 |
| M01-003 | 登录成功后保存加密 Token | 登录成功 | 读取配置文件 | `miHome.token` 已加密存储 | P0 |
| M01-004 | 会话缺失时禁止同步 | 未登录 | 调用 `syncFromCloud()` | 返回 `SESSION_MISSING` | P0 |
| M01-005 | 会话过期时提示重新登录 | Token 过期 | 调用 `syncFromCloud()` | 返回 `SESSION_EXPIRED` | P0 |

### 云端同步测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|

## 发现的问题

1. `Control Context` 的来源仍依赖后续控制适配层补充，测试中需要接受“设备已同步但不可控”的结果。
2. 会话刷新策略目前只定义了错误出口，没有完整自动刷新细节，编码时需要补齐实现约束。

## 建议

1. 先用 mock 适配层稳定登录和同步测试，再引入真实云端联调。
2. 将“同步成功但部分设备不可控”纳入正常首期验收标准。
