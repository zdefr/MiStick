# 设备管理模块测试用例 (M01)

## 测试范围

设备管理模块负责米家设备的云端同步、信息管理和设备缓存，为设备控制模块提供设备数据支持。

**版本范围：** v0.1 (云端同步、设备信息管理、本地缓存)

## 发现的问题

1. **设备类型识别逻辑缺失** - 设计文档提到 `yeelink.light.*` 映射到 Light，但未说明未知 model 的处理策略
2. **Token 刷新机制缺失** - Token 过期后是否需要用户重新登录？设计未提及自动刷新策略
3. **房间管理接口不一致** - v0.1 说"不做房间管理"，但详细设计中有 `addRoom`/`removeRoom` 方法（标记为 v1.0）

## 建议

1. 添加未知 deviceType 的降级处理，统一归类为 Unknown 类型
2. 补充 Token 刷新机制设计，明确过期后用户流程
3. 在 v0.1 版本中移除房间管理相关接口，保持设计一致
