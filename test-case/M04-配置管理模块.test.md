# 配置管理模块测试用例 (M04)

## 测试范围

配置管理模块负责配置加载、校验、保存、备份恢复、文件权限收敛，以及 `miHome.token` 的加密存储。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```
配置管理模块测试用例
├── ConfigManager 测试
│   ├── ✓ load 读取配置并解密 token
│   ├── ✓ save 保存配置并更新时间
│   ├── ✓ set/get 支持嵌套路径
│   ├── ✓ reset 恢复默认配置
│   └── ✓ backup/restore 工作正常
├── CryptoService 测试
│   ├── ✓ encryptToken/decryptToken
│   ├── ✓ 错误密文抛出异常
│   └── ✓ 相同明文加密结果不同
├── 配置校验测试
│   ├── ✓ 必填字段校验
│   ├── ✓ opacity 范围校验
│   ├── ✓ refreshInterval 范围校验
│   └── ✓ theme 枚举校验
└── 文件权限测试
    ├── ✓ 写入后执行权限收敛
    └── ✓ 权限设置失败时记录警告
```

## 发现的问题

1. 文件权限在不同平台上的实现方式可能不同，测试应关注“权限收敛行为”而不是单一系统调用。
2. `KeyProvider` 的实际实现未在设计中展开，编码时需要补足测试替身方案。

## 建议

1. 对加密服务和权限设置都使用可注入依赖，便于单元测试。
2. 将“token 加密”和“配置文件可编辑”作为同时成立的验收条件。
