# 设备控制模块测试用例 (M02)

## 测试范围

设备控制模块负责对可控设备执行状态查询和控制指令下发，并依赖“云控优先、本地回退”的统一控制链路完成实际设备调用。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```text
设备控制模块测试用例
├── DeviceController 测试
│   ├── ✓ turnOn/turnOff 成功控制
│   ├── ✓ setBrightness 范围校验
│   ├── ✓ getStatus 查询设备状态
│   ├── ✓ 云控失败后本地回退
│   └── ✓ 不可控设备返回 DEVICE_NOT_CONTROLLABLE
├── CloudControlService 测试
│   ├── ✓ 调用 /api/cloud/control
│   ├── ✓ 调用 /api/cloud/status
│   └── ✓ 健康检查失败返回不可用
├── LocalControlService 测试
│   ├── ✓ 调用 /api/control
│   ├── ✓ 调用 /api/status
│   └── ✓ 仅在回退时被调用
├── IPC 测试
│   ├── ✓ device:control
│   ├── ✓ device:getStatus
│   └── ✓ device:batchControl
└── 性能与异常测试
    ├── ✓ 单次云控响应 < 3 秒
    ├── ✓ 本地回退响应 < 2 秒
    ├── ✓ TimeoutError 按策略重试
    └── ✓ ControlRouteNotFound 直接失败
```

## 详细测试用例

### DeviceController 测试

| 测试 ID | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 |
|---------|----------|----------|----------|----------|--------|
| M02-001 | 云控成功控制设备 | 设备存在、具备云控能力 | 调用 `turnOn(deviceId)` | 返回成功结果且 `route=cloud` | P0 |
| M02-002 | 云控失败后本地回退成功 | 设备同时具备云控和本地能力 | mock 云控失败后调用 `turnOn(deviceId)` | 返回成功结果且 `route=local` | P0 |
| M02-003 | 不存在设备返回错误 | deviceId 无效 | 调用 `turnOn(deviceId)` | 返回 `DEVICE_NOT_FOUND` | P0 |
| M02-004 | 不可控设备返回错误 | 无控制能力 | 调用 `turnOn(deviceId)` | 返回 `DEVICE_NOT_CONTROLLABLE` | P0 |
| M02-005 | 无可用控制路径返回错误 | 云控和本地都不可用 | 调用 `turnOn(deviceId)` | 返回 `CONTROL_ROUTE_NOT_FOUND` | P0 |
| M02-006 | 亮度设置成功 | 灯设备可控 | 调用 `setBrightness(deviceId, 80)` | 返回成功结果并更新本地状态 | P0 |
| M02-007 | 亮度越界校验失败 | 灯设备可控 | 调用 `setBrightness(deviceId, 150)` | 返回 `INVALID_PARAMS` | P0 |
| M02-008 | 状态查询优先走云控 | 设备具备云控能力 | 调用 `getStatus(deviceId)` | 返回最新状态 | P0 |
| M02-009 | 状态查询回退本地 | 云端状态查询失败且本地可用 | 调用 `getStatus(deviceId)` | 返回本地状态 | P1 |
| M02-010 | 批量控制返回聚合结果 | 多设备可控 | 调用 `batchControl(commands)` | 返回 `CommandResult[]` | P1 |

## 性能测试用例

| 测试 ID | 测试场景 | 性能指标 | 测试方法 | 预期结果 |
|---------|----------|----------|----------|----------|
| M02-PERF-001 | 单次云控响应 | < 3 秒 | 调用 `turnOn()` 并计时 | 响应时间 < 3 秒 |
| M02-PERF-002 | 本地回退响应 | < 2 秒 | 模拟云控失败后回退 | 响应时间 < 2 秒 |
| M02-PERF-003 | 批量控制 (10 设备) | < 5 秒 | 调用 `batchControl(10 个命令)` | 完成时间 < 5 秒 |

## 发现的问题

1. 部分设备首期可能只有云控能力，没有本地回退能力，测试需要接受这种不对称能力结构。
2. 第三方库选择仍属于实现细节，测试应围绕统一接口和路由策略编写。

## 建议

1. 优先建设 mock 云控桥接服务，再补本地回退场景。
2. 把“云控失败后本地回退”纳入核心回归集。
