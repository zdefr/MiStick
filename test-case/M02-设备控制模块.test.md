# 设备控制模块测试用例 (M02)

## 测试范围

设备控制模块负责对可控设备执行单设备控制、状态查询和错误转换，并依赖本地控制适配层完成实际设备调用。

**版本范围：** v0.1 首期基线

## 测试用例思维导图

```
设备控制模块测试用例
├── DeviceController 测试
│   ├── ✓ turnOn/turnOff 成功控制
│   ├── ✓ setBrightness 范围校验
│   ├── ✓ getStatus 查询设备状态
│   ├── ✓ 不可控设备返回 DEVICE_NOT_CONTROLLABLE
│   └── ✓ 缺少 controlContext 返回 CONTROL_CONTEXT_MISSING
├── DeviceControlService 测试
│   ├── ✓ 调用 /api/control
│   ├── ✓ 调用 /api/status
│   └── ✓ 健康检查失败返回不可用
├── IPC 测试
│   ├── ✓ device:turnOn
│   ├── ✓ device:turnOff
│   ├── ✓ device:setBrightness
│   ├── ✓ device:getStatus
│   └── ✓ device:batchControl
└── 性能与异常测试
    ├── ✓ 单次控制响应 < 2 秒
    ├── ✓ 10 设备批量控制 < 5 秒
    ├── ✓ TimeoutError 最多重试 2 次
    └── ✓ ControlServiceUnavailable 不自动重试
```

## 发现的问题

1. 部分设备在首期可能仅能展示、不能控制，测试需要区分“同步成功”和“控制成功”两个概念。
2. 本地控制适配层的协议与第三方库选择仍是实现细节，测试应围绕统一接口而非库细节编写。

## 建议

1. 优先建设 mock 控制服务，先跑通 Electron 侧主流程。
2. 把设备不可控和控制上下文缺失纳入核心回归集。
