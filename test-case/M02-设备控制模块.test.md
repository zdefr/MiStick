# 设备控制模块测试用例 (M02)

## 测试范围

设备控制模块负责向米家设备发送控制指令，查询设备状态，处理控制结果反馈。

## 发现的问题

1. **状态同步策略模糊** - 控制成功后"更新本地状态"，但未说明是轮询还是依赖设备上报
2. **命令队列设计过度** - 设计中有 `CommandQueue` 防止并发冲突，但实际米家设备支持并发控制
3. **Python 服务健康检查缺失** - 未设计 Python 设备服务的健康检查/自动重启机制

## 建议

1. 明确状态同步策略：控制成功后主动查询一次设备状态确认
2. 简化命令队列设计，仅在必要时串行化同一设备的控制指令
3. 添加 Python 服务健康检查机制，服务异常时自动重启
