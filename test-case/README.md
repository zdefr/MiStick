## 设计问题汇总

在测试用例设计过程中发现以下问题，需要在开发前澄清或优化：

### 跨模块共性问题

| 问题 | 影响模块 | 建议 |
|------|----------|------|
| 配置版本迁移缺失 | M04 | 添加配置版本迁移函数 |
| 并发写入风险 | M04, M07 | 添加写入队列或锁机制 |
| 持久化策略缺失 | M06, M07 | 明确数据持久化位置 |
| 安全性考虑不足 | M04, M06 | Token 加密、脚本签名验证 |

### 各模块特定问题

| 模块 | 问题 | 建议 |
|------|------|------|
| M01 | 设备类型识别逻辑缺失 | 添加未知 model 降级处理 |
| M01 | Token 刷新机制缺失 | 补充自动刷新策略 |
| M02 | 状态同步策略模糊 | 明确轮询或上报机制 |
| M03 | AgentInput 定位模糊 | v0.1 隐藏该组件 |
| M05 | WebSocket 依赖过强 | 添加 HTTP 轮询降级 |
| M06 | Python 路径硬编码 | 支持用户配置 Python 路径 |

## 快速开始

### 运行所有测试

```bash
# 前端测试
npm run test

# Python 服务测试
cd python-service && pytest

# 生成覆盖率报告
npm run test:coverage
```

### 运行单个模块测试

```bash
# 配置管理模块
npm run test -- config

# 设备管理模块
npm run test -- device
```

## 文档说明

每个模块的测试用例文件包含：

1. **测试用例思维导图** - 树状结构展示测试范围
2. **详细测试用例表** - 包含测试 ID、场景、步骤、预期结果
3. **性能测试用例** - 性能指标和验收标准
4. **发现的问题** - 设计中的问题和优化建议
5. **建议** - 针对问题的改进建议
