## 2. 产品概述

### 2.1 产品定位

一款运行在 Windows 桌面上的轻量级米家设备控制工具，以**便利贴**形式常驻桌面，提供快速、直观的设备操作入口。支持通过 WSL 上的 **OpenClaw** 实现智能化、自动化控制。

### 2.2 目标用户

- 拥有米家智能家居设备的 Windows 用户
- 希望在桌面快速控制设备，无需打开完整米家 App
- 追求简洁、高效操作体验的用户
- 有智能化、自动化控制需求的进阶用户

### 2.3 核心价值

| 价值点 | 描述 |
|--------|------|
| **快捷** | 桌面常驻，一键触达，无需切换窗口 |
| **简洁** | 仅保留核心设备控制功能，无冗余信息 |
| **直观** | 便利贴形态，视觉轻量，操作直观 |
| **智能** | 支持 OpenClaw 规则引擎，实现自动化场景联动 |

## 3. 功能需求

### 3.1 功能范围

**本期包含（MVP）：**

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 设备连接 | 米家账号登录 | P0 |

| 设备控制 | 开关类设备控制（灯、插座等） | P0 |

| 界面展示 | 便利贴形态展示 | P0 |

### 4.4 OpenClaw 集成方案

**OpenClaw 简介：**

OpenClaw 是一个本地部署的开源、高可扩展的 AI Agent 框架，支持：
- 自然语言理解
- 工具/函数调用（Tool Calling）
- 自动化任务编排
- 本地 LLM 支持

**定位说明：**

OpenClaw 在本项目中作为**可选的批处理增强模块**，用于：
- 执行复杂场景联动（如"回家模式"：同时开灯、开空调、开电视）
- 自然语言交互（如"我要睡觉了"→ 关闭所有设备）
- 定时/条件触发批处理任务

**WSL 部署方式：**

```bash
# 在 WSL 中安装 OpenClaw
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pip install -r requirements.txt
openclaw start
```

**工具注册（Tool Registration）：**

OpenClaw 通过注册 Windows 批处理脚本作为可调用工具：

```yaml
# openclaw.yaml
tools:
  - name: home_mode
    description: 回家模式 - 开灯、开空调、开电视
    command: python C:\mijia-scripts\home_mode.py
    
  - name: sleep_mode
    description: 睡眠模式 - 关闭所有设备
    command: python C:\mijia-scripts\sleep_mode.py
```

**通信方式：**

| 方式 | 描述 |
|------|------|
| HTTP API | OpenClaw 提供 RESTful API 接收指令和查询状态 |
| WebSocket | 支持实时推送 Agent 执行状态 |
| 工具调用 | Agent 解析用户意图后调用注册的批处理脚本 |

## 8. 风险与依赖

| 风险 | 描述 | 应对措施 |
|------|------|----------|
| API 稳定性 | 小米 IoT API 可能变更 | 使用成熟开源库，关注社区更新 |
| 设备兼容性 | 部分设备协议不支持 | 优先支持主流设备类型，逐步扩展 |
| 账号安全 | 登录态泄露风险 | Token 加密存储，提供手动登出 |
| WSL 兼容性 | WSL 网络配置复杂 | 提供详细的 WSL 配置指南，支持手动配置地址 |
| OpenClaw 依赖 | OpenClaw 项目维护状态 | 评估项目活跃度，保持配置层可插拔 |
| Agent 准确性 | AI 意图识别可能不准确 | 提供确认机制，支持手动修正 |

from miio import Device

def home_mode():
    """执行回家模式：开灯、开空调、开电视"""
    devices = {
        "客厅灯": Device("192.168.1.101", "token_xxx"),
        "空调": Device("192.168.1.102", "token_xxx"),
        "电视": Device("192.168.1.103", "token_xxx"),
    }
    
    results = []
    for name, dev in devices.items():
        dev.send("set_power", ["on"])
        results.append(f"✅ {name}: 已打开")
    
    print("\n".join(results))
    return {"status": "success", "details": results}

if __name__ == "__main__":
    home_mode()
