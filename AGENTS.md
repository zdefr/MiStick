# 米家桌面便利贴 - Codex 接手说明

## 项目现状

- 当前仓库仍处于文档阶段，`packages/` 为空，尚未开始正式开发。
- 当前开发基线以 `PRD.md`、`HLD.md`、`docs/设备管理模块详细设计.md`、`docs/设备控制模块详细设计.md`、`docs/UI 展示模块详细设计.md`、`docs/配置管理模块详细设计.md` 为准。
- `.qwen/settings.json` 目前只有空的 `mcpServers` 配置，没有需要迁移的实际 MCP 实例。

## 当前确认的范围

- 首期必须做：米家账号登录、云端同步设备、基础设备控制、便利贴 UI、配置管理。
- 首期不做：OpenClaw 智能控制、批处理脚本管理、自然语言交互。
- 配置文件本身允许用户修改，但 `miHome.token` 需要加密存储。
- 配置文件权限需要收敛为当前用户可读写。

## 文档权威顺序

1. `PRD.md`
2. `HLD.md`
3. `docs/*.md`
4. `test-case/*.md`
5. `memory/2026-03-17.log`

如文档冲突，优先以上层文档为准，并同步修正文档后再落代码。

## 实施前注意事项

- M01 的首期目标是云端设备同步与缓存，不要求一次性解决所有设备的控制上下文。
- M02 可继续沿用本地控制适配层思路，但要接受部分设备“已同步但暂不可控”的首期状态。
- 不要再把 OpenClaw 配置、Agent Store、脚本管理界面混入首期实现。
- 设计文档中的接口抽象优先于示意性第三方 URL，具体接入细节落在适配层实现。

## 建议实现顺序

1. 搭建 Electron + React + TypeScript 工程骨架。
2. 先做 M04 配置管理模块。
3. 再做 M01 米家登录与云端同步。
4. 接着做 M02 基础控制链路。
5. 最后完成 M03 便利贴 UI 联调和 M07 日志补充。
