## 2. 模块概述

### 2.1 职责

配置管理模块负责应用配置的存储、读取、更新、验证和备份。配置文件保持用户可读可改，敏感字段仅对米家登录 Token 做加密存储。

### 2.2 首期范围

**首期必须实现：**
- 启动时加载配置
- 配置校验与默认值补齐
- 配置项读写
- 配置备份与恢复
- 米家登录 Token 加密存储
- 配置文件权限收敛到当前用户可读写

**首期不做：**
- OpenClaw 配置管理
- 批处理脚本配置管理
- 多账号切换策略

### 2.3 功能范围

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 配置读取 | 启动时加载配置 | P0 |
| 配置写入 | 变更时保存配置 | P0 |
| 配置验证 | 校验结构和字段范围 | P0 |
| 配置备份 | 保存变更前快照 | P1 |
| 配置恢复 | 从备份恢复 | P1 |
| Token 加密 | 对米家登录 Token 加密存储 | P0 |
| 文件权限设置 | 控制配置文件为当前用户可读写 | P0 |

## 3. 配置结构

### 3.1 配置文件结构

```typescript
interface AppConfig {
  version: string;
  userId: string;
  miHome: {
    accountId?: string;
    token?: string;              // 加密后存储
    tokenExpiry?: string;
    region: 'cn' | 'de' | 'us';
  };
  window: {
    width: number;
    height: number;
    x?: number;
    y?: number;
    alwaysOnTop: boolean;
    opacity: number;             // 0.1 - 1.0
    skipTaskbar: boolean;
  };
  appearance: {
    theme: 'light' | 'dark' | 'system';
    fontSize: number;
    language: 'zh-CN' | 'en';
  };
  devices: {
    autoRefresh: boolean;
    refreshInterval: number;     // 秒
    lastSyncAt?: string;
  };
  logging: {
    level: 'debug' | 'info' | 'warn' | 'error';
    maxFiles: number;
    maxSize: string;
  };
  createdAt: string;
  updatedAt: string;
}
```

说明：
- 配置文件是面向用户的可编辑文件，非敏感字段保持明文。
- `miHome.token` 是当前首期唯一要求加密存储的字段。
- OpenClaw 和脚本配置不进入当前配置结构。

### 4.2 核心类说明

#### 4.2.1 ConfigManager

```typescript
class ConfigManager {
  private configPath = path.join(app.getPath('userData'), 'config.json');
  private backupDir = path.join(app.getPath('userData'), 'backups');
  private config: AppConfig | null = null;
  private cryptoService = new CryptoService();

  async load(): Promise<AppConfig> {
    const raw = await fs.readFile(this.configPath, 'utf-8');
    const config = JSON.parse(raw) as AppConfig;

    if (config.miHome.token) {
      config.miHome.token = await this.cryptoService.decryptToken(config.miHome.token);
    }

    this.validate(config);
    this.config = config;
    return config;
  }

  async save(patch?: Partial<AppConfig>): Promise<void> {
    const nextConfig = mergeConfig(this.config ?? this.getDefaultConfig(), patch ?? {});
    this.validate(nextConfig);

    const toSave = structuredClone(nextConfig);
    if (toSave.miHome.token) {
      toSave.miHome.token = await this.cryptoService.encryptToken(toSave.miHome.token);
    }

    toSave.updatedAt = new Date().toISOString();
    await this.ensureBackupBeforeWrite();
    await fs.writeFile(this.configPath, JSON.stringify(toSave, null, 2), 'utf-8');
    await this.ensureFilePermission();
    this.config = nextConfig;
  }
}
```

#### 4.2.2 CryptoService

```typescript
class CryptoService {
  private algorithm = 'aes-256-gcm';
  private keyProvider = new KeyProvider();

  async encryptToken(plainText: string): Promise<string> {
    const key = await this.keyProvider.getOrCreateKey();
    const iv = crypto.randomBytes(12);
    const cipher = crypto.createCipheriv(this.algorithm, key, iv);
    const encrypted = Buffer.concat([cipher.update(plainText, 'utf8'), cipher.final()]);
    const authTag = cipher.getAuthTag();
    return Buffer.concat([iv, authTag, encrypted]).toString('base64');
  }

  async decryptToken(cipherText: string): Promise<string> {
    const key = await this.keyProvider.getOrCreateKey();
    const payload = Buffer.from(cipherText, 'base64');
    const iv = payload.subarray(0, 12);
    const authTag = payload.subarray(12, 28);
    const encrypted = payload.subarray(28);
    const decipher = crypto.createDecipheriv(this.algorithm, key, iv);
    decipher.setAuthTag(authTag);
    return Buffer.concat([decipher.update(encrypted), decipher.final()]).toString('utf8');
  }
}
```

说明：
- 配置文件权限与 Token 加密是两件不同的事。
- 文件权限控制的是 `config.json` 仅允许当前用户读写。
- Token 加密只作用于 `miHome.token` 字段，避免将登录态明文写入配置。

## 6. 流程设计

### 6.1 配置加载流程

```
应用启动
  ↓
ConfigManager.load()
  ↓
读取 config.json
  ↓
存在 miHome.token ?
  ├─ 否：直接校验配置
  └─ 是：解密 token 后再校验配置
  ↓
返回内存态配置对象
```

### 6.2 配置更新流程

```
用户修改配置
  ↓
ConfigManager.save(patch)
  ↓
合并默认值与 patch
  ↓
校验字段合法性
  ↓
仅对 miHome.token 加密
  ↓
写入配置文件
  ↓
收敛文件权限
```

### 6.3 备份流程

```
写入配置前
  ↓
生成时间戳备份文件
  ↓
复制当前 config.json 到 backups/
  ↓
写入新配置
  ↓
清理过期备份
```

### 7.2 配置文件权限

```typescript
async function ensureFilePermission(configPath: string): Promise<void> {
  // 目标：配置文件仅允许当前用户读写。
  // 在支持 POSIX 权限位的环境可使用 chmod；
  // 在 Windows 环境中由实现层映射为等价权限控制。
  await chmod(configPath, 0o600);
}
```

说明：
- 权限控制针对整个配置文件。
- 配置文件仍允许用户手工修改，只要该用户具备读写权限。
- 权限控制不替代 Token 加密，两者同时存在。

## 8. 安全设计

### 8.1 Token 加密范围

当前首期仅对以下字段加密：
- `miHome.token`

以下字段保持明文，便于用户查看和手工编辑：
- `miHome.region`
- `window.*`
- `appearance.*`
- `devices.*`
- `logging.*`

### 8.2 密钥管理原则

- 加密密钥由 `KeyProvider` 统一提供，不散落在业务代码中。
- 解密失败时，不回退到明文逻辑，直接抛出 `DecryptFailed`。
- 业务层只能调用 `encryptToken/decryptToken`，不直接操作算法细节。

### 8.3 配置校验规则

| 字段 | 规则 |
|------|------|
| `window.opacity` | 0.1 - 1.0 |
| `window.width` | >= 320 |
| `window.height` | >= 400 |
| `devices.refreshInterval` | 30 - 3600 秒 |
| `appearance.fontSize` | 12 - 24 |

## 9. 异常处理

### 9.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| ConfigNotFound | 配置文件不存在 | 创建默认配置 |
| ConfigCorrupted | JSON 解析失败或结构非法 | 恢复备份或重建默认配置 |
| DecryptFailed | Token 解密失败 | 清空登录态并提示重新登录 |
| ValidationFailed | 字段不符合规则 | 拒绝写入并提示用户 |
| WriteFailed | 文件写入失败 | 重试并记录日志 |
| PermissionFailed | 权限设置失败 | 记录警告并继续运行 |

### 9.2 错误码定义

```typescript
enum ConfigErrorCode {
```
