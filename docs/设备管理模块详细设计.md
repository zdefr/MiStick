## 2. 模块概述

### 2.1 职责

设备管理模块负责米家设备的云端同步、信息管理和设备缓存，为设备控制模块提供设备数据支持。

**第一期简化说明：**
- ❌ 不做：本地设备发现、手动添加设备（通过配置文件或云端同步）
- ✅ 要做：云端同步设备、设备信息管理、设备缓存

### 2.2 功能范围

| 功能 | 说明 | 优先级 | 版本 |
|------|------|--------|------|
| 云端同步 | 从米家云端同步设备列表 | P0 | v0.1 |
| 设备信息管理 | 设备名称、房间、备注等 | P0 | v0.1 |
| 设备缓存 | 本地缓存设备信息 | P0 | v0.1 |
| 设备分组 | 按房间/类型分组 | P1 | v1.0 |
| 配置导入 | 从配置文件加载预设设备 | P2 | v2.0 |
| 设备发现 | 扫描局域网内的米家设备 | P2 | v2.0 |
| 设备添加 | 手动添加设备到管理列表 | P2 | v2.0 |
| 设备删除 | 移除设备 | P2 | v2.0 |

**第一期 (v0.1) 功能范围：**
- 从米家云端同步设备列表
- 支持修改设备名称、房间归属
- 本地缓存设备信息

#### 3.2.2 Device

设备实体类。

```typescript
interface Device {
  id: string;           // 设备 ID (DID)
  name: string;         // 设备名称
  type: DeviceType;     // 设备类型
  model: string;        // 设备型号
  ip: string;           // IP 地址
  token: string;        // 设备 Token
  roomId?: string;      // 所属房间 ID
  status: DeviceStatus; // 当前状态
  capabilities: string[]; // 支持的能力
  createdAt: Date;      // 创建时间
  updatedAt: Date;      // 更新时间
}
```

enum DeviceType {

## 4. 接口设计

### 4.1 内部接口 (IPC)

#### 4.1.1 获取设备列表

```typescript
// 渲染进程 → 主进程
ipcRenderer.invoke('device:getAll')

// 主进程处理
ipcMain.handle('device:getAll', async () => {
  return deviceManager.getAllDevices();
});
```

#### 4.1.2 云端同步

```typescript
// 请求参数
interface SyncFromCloudRequest {
  force?: boolean;  // 是否强制刷新
}

// 调用 (v0.1)
ipcRenderer.invoke('device:syncFromCloud', { force: false });

// 主进程处理
ipcMain.handle('device:syncFromCloud', async (event, { force }) => {
  const devices = await deviceManager.syncFromCloud(force);
  return devices;
});

// 返回
interface SyncFromCloudResponse {
  devices: Device[];
  syncedAt: string;
}
```

#### 4.1.3 更新设备

```typescript
// 请求参数
interface UpdateDeviceRequest {
  deviceId: string;
  updates: Partial<Device>;
}

// 调用
ipcRenderer.invoke('device:update', { deviceId, updates });

// 主进程处理
ipcMain.handle('device:update', async (event, { deviceId, updates }) => {
  await deviceManager.updateDevice(deviceId, updates);
});
```

## 6. 数据存储设计

### 6.1 SQLite 表结构

#### 6.1.1 devices 表

```sql
CREATE TABLE devices (
  id TEXT PRIMARY KEY,              -- 设备 ID (DID)
  name TEXT NOT NULL,               -- 设备名称
  type TEXT NOT NULL,               -- 设备类型
  model TEXT,                       -- 设备型号
  ip TEXT,                          -- IP 地址
  token TEXT,                       -- 设备 Token
  room_id TEXT,                     -- 所属房间 ID
  capabilities TEXT,                -- 能力列表 (JSON)
  status TEXT,                      -- 状态 (JSON)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_room ON devices(room_id);
CREATE INDEX idx_devices_type ON devices(type);
```

#### 6.1.2 rooms 表

```sql
CREATE TABLE rooms (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  device_ids TEXT,                  -- 设备 ID 列表 (JSON)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 缓存策略

| 数据类型 | 缓存策略 | 过期时间 |
|----------|----------|----------|
| 设备列表 | 启动时加载，变更时更新 | 无过期 |
| 设备状态 | 按需加载，定期刷新 | 5 分钟 |
| 房间信息 | 启动时加载 | 无过期 |

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| DeviceNotFound | 设备 ID 不存在 | 返回 null，UI 提示 |
| DeviceOffline | 设备离线 | 标记状态为离线，提示用户 |
| TokenInvalid | Token 无效 | 提示重新获取 Token |
| NetworkError | 网络错误 | 重试 3 次，失败后提示 |
