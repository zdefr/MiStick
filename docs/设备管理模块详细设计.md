## 2. 模块概述

### 2.1 职责

设备管理模块负责米家账号登录后的设备云端同步、设备元数据管理和本地缓存维护，为 UI 展示模块提供设备列表和分组数据。

### 2.2 首期范围

**首期必须实现：**
- 米家登录态读取与校验
- 从米家云端同步设备列表
- 从米家云端同步房间/家庭信息
- 本地缓存设备元数据
- 启动时基于缓存和登录态决定是否自动同步

**首期明确不做：**
- 局域网扫描发现设备
- 手动添加/删除设备
- 从配置文件导入设备
- OpenClaw 相关能力

### 2.3 功能范围

| 功能 | 说明 | 优先级 | 版本 |
|------|------|--------|------|
| 账号会话读取 | 从配置管理模块获取米家登录态 | P0 | v0.1 |
| 云端同步 | 从米家云端同步设备和房间信息 | P0 | v0.1 |
| 设备缓存 | 使用 SQLite 缓存设备元数据 | P0 | v0.1 |
| 设备信息更新 | 更新本地备注、房间映射等展示信息 | P0 | v0.1 |
| 房间分组 | 按房间输出设备列表 | P1 | v1.0 |
| 手动发现/添加/删除 | 后续迭代再设计 | - | - |

### 3.2 核心类说明

#### 3.2.1 DeviceManager

设备管理模块主控类。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `login` | `credentials` | `Promise<LoginResult>` | 执行米家登录并保存登录态 |
| `syncFromCloud` | `force?: boolean` | `Promise<SyncResult>` | 从米家云端同步设备和房间 |
| `getAllDevices` | - | `Device[]` | 获取当前缓存设备列表 |
| `getDevicesByRoom` | `roomId` | `Device[]` | 按房间获取设备 |
| `getRooms` | - | `Room[]` | 获取房间列表 |
| `updateDeviceMeta` | `deviceId, updates` | `Promise<void>` | 更新本地展示信息 |
| `loadFromCache` | - | `Promise<void>` | 从 SQLite 预热缓存 |

#### 3.2.2 MiHomeCloudService

米家云端同步适配层，不在本模块写死第三方接口细节，只定义能力边界。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `login` | `credentials` | `Promise<SessionInfo>` | 登录并返回会话 |
| `refreshSession` | `session` | `Promise<SessionInfo>` | 刷新登录态 |
| `getHomes` | `session` | `Promise<Home[]>` | 获取家庭列表 |
| `getRooms` | `session, homeId` | `Promise<Room[]>` | 获取房间列表 |
| `getDevices` | `session, homeId` | `Promise<CloudDevice[]>` | 获取设备列表 |

#### 3.2.3 DeviceRepository

设备缓存仓储层，负责 SQLite 读写。

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `findAllDevices` | `Promise<Device[]>` | 读取全部设备 |
| `replaceDevices` | `Promise<void>` | 使用同步结果覆盖缓存 |
| `replaceRooms` | `Promise<void>` | 更新房间缓存 |
| `updateDeviceMeta` | `Promise<void>` | 更新本地备注/房间映射 |

### 3.3 数据结构

```typescript
interface LoginCredentials {
  username: string;
  password: string;
  region: 'cn' | 'de' | 'us';
}

interface SessionInfo {
  accountId: string;
  token: string;
  tokenExpiry?: string;
  region: 'cn' | 'de' | 'us';
}

interface Device {
  id: string;
  name: string;
  model: string;
  type: DeviceType;
  homeId: string;
  roomId?: string;
  roomName?: string;
  online?: boolean;
  isControllable: boolean;
  controlContext?: {
    ip?: string;
    token?: string;
    protocol?: string;
  };
  extra?: Record<string, unknown>;
  syncedAt: string;
  createdAt: string;
  updatedAt: string;
}

interface Room {
  id: string;
  homeId: string;
  name: string;
}

interface SyncResult {
  devices: Device[];
  rooms: Room[];
  syncedAt: string;
  fromCache: boolean;
}

enum DeviceType {
  Light = 'light',
  Switch = 'switch',
  Socket = 'socket',
  Curtain = 'curtain',
  Sensor = 'sensor',
  Unknown = 'unknown'
}
```

说明：
- `controlContext` 在首期是可选字段，不作为云端同步成功的前置条件。
- 首期同步目标是“设备元数据完整”，不是“所有设备立即可控”。

## 4. 接口设计

### 4.1 内部接口 (IPC)

#### 4.1.1 米家登录

```typescript
interface LoginRequest {
  username: string;
  password: string;
  region: 'cn' | 'de' | 'us';
}

ipcRenderer.invoke('auth:login', {
  username,
  password,
  region: 'cn',
});

ipcMain.handle('auth:login', async (_event, payload: LoginRequest) => {
  return deviceManager.login(payload);
});
```

#### 4.1.2 获取设备列表

```typescript
ipcRenderer.invoke('device:getAll');

ipcMain.handle('device:getAll', async () => {
  return deviceManager.getAllDevices();
});
```

#### 4.1.3 云端同步

```typescript
interface SyncFromCloudRequest {
  force?: boolean;
}

ipcRenderer.invoke('device:syncFromCloud', { force: false });

ipcMain.handle('device:syncFromCloud', async (_event, { force }) => {
  return deviceManager.syncFromCloud(force);
});
```

#### 4.1.4 获取房间列表

```typescript
ipcRenderer.invoke('device:getRooms');

ipcMain.handle('device:getRooms', async () => {
  return deviceManager.getRooms();
});
```

#### 4.1.5 更新本地设备展示信息

```typescript
interface UpdateDeviceMetaRequest {
  deviceId: string;
  updates: {
    name?: string;
    roomId?: string;
    roomName?: string;
  };
}

ipcRenderer.invoke('device:updateMeta', { deviceId, updates });
```

### 4.2 外部接口 (云端同步抽象)

本模块不在详细设计中固化第三方 HTTP URL，而是抽象为以下能力：

| 能力 | 请求 | 返回 |
|------|------|------|
| 登录 | `LoginCredentials` | `SessionInfo` |
| 刷新登录态 | `SessionInfo` | `SessionInfo` |
| 获取家庭列表 | `SessionInfo` | `Home[]` |
| 获取房间列表 | `SessionInfo + homeId` | `Room[]` |
| 获取设备列表 | `SessionInfo + homeId` | `CloudDevice[]` |

适配层实现要求：
- 将第三方返回映射为模块内部统一结构。
- 处理登录失效、频控、重试和错误码转换。
- 对外仅暴露 Promise 风格接口，不把第三方 SDK/响应对象泄露到业务层。

### 5.3 启动自动同步流程

```
应用启动
  ↓
DeviceManager.loadFromCache()
  ↓
检查 miHome.token 是否存在且未过期
  ↓
有有效登录态？
  ├─ 否：返回缓存并提示登录
  └─ 是：按缓存年龄判断是否触发 syncFromCloud()
```

缓存策略：
- 启动先读缓存，保证界面尽快可见。
- 当存在有效登录态且缓存超过 10 分钟时，后台自动触发同步。
- 用户手动点击刷新时，始终强制同步。

## 6. 数据存储设计

### 6.1 SQLite 表结构

#### 6.1.1 devices 表

```sql
CREATE TABLE devices (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  model TEXT NOT NULL,
  type TEXT NOT NULL,
  home_id TEXT NOT NULL,
  room_id TEXT,
  room_name TEXT,
  online INTEGER,
  is_controllable INTEGER NOT NULL DEFAULT 0,
  control_context TEXT,
  extra TEXT,
  synced_at DATETIME NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_home_id ON devices(home_id);
CREATE INDEX idx_devices_room_id ON devices(room_id);
CREATE INDEX idx_devices_type ON devices(type);
```

#### 6.1.2 rooms 表

```sql
CREATE TABLE rooms (
  id TEXT PRIMARY KEY,
  home_id TEXT NOT NULL,
  name TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_rooms_home_id ON rooms(home_id);
```

### 6.2 缓存策略

| 数据类型 | 策略 | 说明 |
|----------|------|------|
| 设备列表 | 全量替换 | 每次同步后以云端结果为准 |
| 房间列表 | 全量替换 | 与设备同步同批更新 |
| 本地展示信息 | 局部更新 | 用户修改备注/房间时更新 |
| 控制上下文 | 可选保留 | 若后续适配层补充则写入 `control_context` |

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| SessionMissing | 未登录或无可用会话 | 提示用户登录 |
| SessionExpired | 登录态过期 | 触发重新登录 |
| SyncFailed | 云端同步失败 | 保留旧缓存并提示重试 |
| CacheWriteFailed | 缓存写入失败 | 记录日志并回退本次更新 |
| MappingFailed | 云端结构映射失败 | 记录错误样本并跳过异常项 |

### 7.2 错误码定义

```typescript
enum DeviceManageErrorCode {
  SUCCESS = 0,
  SESSION_MISSING = 1001,
  SESSION_EXPIRED = 1002,
  SYNC_FAILED = 1003,
  CACHE_WRITE_FAILED = 1004,
  MAPPING_FAILED = 1005,
}
```
