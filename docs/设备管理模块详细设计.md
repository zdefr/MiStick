## 2. 模块概述

### 2.1 职责

设备管理模块负责基于 `mijia-api` 完成米家扫码登录、登录态复用、家庭/房间/设备云端同步，以及本地设备缓存维护，为 UI 和 M02 提供统一设备元数据。

### 2.2 首期范围

**首期必须实现：**
- 启动二维码登录流程
- 轮询二维码登录结果
- 基于认证文件复用登录态
- 从米家云端同步家庭、房间和设备列表
- 为设备写入统一 `controlCapability` 与 `controlContext`
- 本地缓存设备元数据

**首期明确不做：**
- 账号密码登录
- 局域网扫描发现设备
- 手动添加/删除设备
- OpenClaw 相关能力

### 2.3 功能范围

| 功能 | 说明 | 优先级 | 版本 |
|------|------|--------|------|
| 二维码登录 | 通过 MiHome Bridge Service 发起扫码登录 | P0 | v0.1 |
| 会话复用 | 复用 `mijia-api` 认证文件，跳过重复扫码 | P0 | v0.1 |
| 云端同步 | 从米家云端同步家庭、房间和设备信息 | P0 | v0.1 |
| 能力映射 | 将云端设备映射为统一设备结构 | P0 | v0.1 |
| 设备缓存 | 使用 SQLite 缓存设备元数据 | P0 | v0.1 |
| 本地展示信息更新 | 更新本地备注、房间映射等展示信息 | P1 | v1.0 |

### 3.2 核心类说明

#### 3.2.1 DeviceManager

设备管理模块主控类。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `startQrLogin` | `region` | `Promise<QrLoginTicket>` | 启动扫码登录 |
| `pollQrLogin` | `ticketId` | `Promise<LoginPollResult>` | 轮询扫码结果 |
| `logout` | - | `Promise<void>` | 清理登录态与本地会话 |
| `syncFromCloud` | `force?: boolean` | `Promise<SyncResult>` | 从米家云端同步设备和房间 |
| `getAllDevices` | - | `Device[]` | 获取当前缓存设备列表 |
| `getDevicesByRoom` | `roomId` | `Device[]` | 按房间获取设备 |
| `getRooms` | - | `Room[]` | 获取房间列表 |
| `updateDeviceMeta` | `deviceId, updates` | `Promise<void>` | 更新本地展示信息 |
| `loadFromCache` | - | `Promise<void>` | 从 SQLite 预热缓存 |

#### 3.2.2 MiHomeBridgeClient

桥接层客户端，封装对 `mijia-api` Python 服务的调用。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `startQrLogin` | `region` | `Promise<QrLoginTicket>` | 创建二维码登录票据 |
| `pollQrLogin` | `ticketId` | `Promise<LoginPollResult>` | 查询登录进度 |
| `logout` | - | `Promise<void>` | 清理桥接层会话 |
| `getHomes` | `session` | `Promise<Home[]>` | 获取家庭列表 |
| `getRooms` | `session, homeId` | `Promise<Room[]>` | 获取房间列表 |
| `getDevices` | `session, homeId` | `Promise<CloudDevice[]>` | 获取设备列表 |

#### 3.2.3 DeviceRepository

设备缓存仓储层，负责 SQLite 读写。

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `findAllDevices` | `Promise<Device[]>` | 读取全部设备 |
| `replaceDevices` | `Promise<void>` | 使用同步结果覆盖缓存 |
| `replaceRooms` | `Promise<void>` | 更新房间缓存 |
| `updateDeviceMeta` | `Promise<void>` | 更新本地备注/房间映射 |

### 3.3 数据结构

```typescript
interface QrLoginTicket {
  ticketId: string;
  qrCodeData: string;     // data URL 或二维码文本
  expiresAt: string;
}

interface LoginPollResult {
  status: 'pending' | 'success' | 'expired' | 'failed';
  session?: SessionInfo;
  errorMessage?: string;
}

interface SessionInfo {
  provider: 'mijia-api';
  accountId?: string;
  region: 'cn' | 'de' | 'us';
  authStoragePath: string;
  lastLoginAt?: string;
}

interface Device {
  id: string;
  name: string;
  model: string;
  type: DeviceType;
  homeId: string;
  roomId?: string;
  roomName?: string;
  online?: boolean;
  controlCapability: {
    cloud: boolean;
    local: boolean;
    preferred: 'cloud' | 'local' | 'none';
  };
  controlContext?: {
    cloud?: {
      did: string;
      props?: Record<string, { siid: number; piid: number }>;
      actions?: Record<string, { siid: number; aiid: number }>;
    };
    local?: {
      ip?: string;
      token?: string;
      protocol?: string;
    };
  };
  extra?: Record<string, unknown>;
  syncedAt: string;
  createdAt: string;
  updatedAt: string;
}

interface Room {
  id: string;
  homeId: string;
  name: string;
}

interface SyncResult {
  devices: Device[];
  rooms: Room[];
  syncedAt: string;
  fromCache: boolean;
}
```

说明：
- `controlCapability.cloud` 表示该设备可通过 `mijia-api` 云控。
- `controlCapability` 仅描述当前已探测到的云控能力与支持动作。
- 首期同步目标是“设备元数据 + 云控能力边界清晰”，不是“所有设备立即支持控制”。

## 4. 接口设计

### 4.1 内部接口 (IPC)

#### 4.1.1 启动二维码登录

```typescript
interface StartQrLoginRequest {
  region: 'cn' | 'de' | 'us';
}

ipcRenderer.invoke('auth:startQrLogin', { region: 'cn' });

ipcMain.handle('auth:startQrLogin', async (_event, payload: StartQrLoginRequest) => {
  return deviceManager.startQrLogin(payload.region);
});
```

#### 4.1.2 轮询二维码登录结果

```typescript
interface PollQrLoginRequest {
  ticketId: string;
}

ipcRenderer.invoke('auth:pollQrLogin', { ticketId });
```

#### 4.1.3 退出登录

```typescript
ipcRenderer.invoke('auth:logout');
```

#### 4.1.4 获取设备列表

```typescript
ipcRenderer.invoke('device:getAll');
```

#### 4.1.5 云端同步

```typescript
interface SyncFromCloudRequest {
  force?: boolean;
}

ipcRenderer.invoke('device:syncFromCloud', { force: false });
```

### 4.2 外部接口 (MiHome Bridge Service)

本模块不在详细设计中固化第三方米家 HTTP URL，而是通过本地桥接服务抽象为以下接口：

| 能力 | 方法 | 请求 | 返回 |
|------|------|------|------|
| 启动扫码登录 | `POST /api/auth/login/start` | `{ region }` | `QrLoginTicket` |
| 轮询扫码结果 | `POST /api/auth/login/poll` | `{ ticketId }` | `LoginPollResult` |
| 登出 | `POST /api/auth/logout` | `{}` | `{ success }` |
| 获取家庭列表 | `GET /api/cloud/homes` | - | `Home[]` |
| 获取房间列表 | `GET /api/cloud/rooms?homeId=...` | - | `Room[]` |
| 获取设备列表 | `GET /api/cloud/devices?homeId=...` | - | `CloudDevice[]` |

适配层实现要求：
- 将第三方返回映射为模块内部统一结构。
- 处理二维码过期、登录失效、频控、重试和错误码转换。
- 对外仅暴露 Promise 风格接口，不把 `mijia-api` 响应对象泄露到业务层。

## 5. 流程设计

### 5.1 扫码登录流程

```text
用户点击登录
  ↓
UI 调用 auth:startQrLogin
  ↓
DeviceManager -> MiHomeBridgeClient.startQrLogin()
  ↓
返回 qrCodeData + ticketId
  ↓
UI 展示二维码并开始轮询 auth:pollQrLogin
  ↓
扫码成功？
  ├─ 否：保持 pending 或提示 expired
  └─ 是：保存 SessionInfo 元数据到配置，结束登录流程
```

### 5.2 云端同步流程

```text
DeviceManager.syncFromCloud()
  ↓
MiHomeBridgeClient.getHomes()
  ↓
按 homeId 拉取 rooms + devices
  ↓
统一映射为 Device / Room
  ↓
写入 SQLite
  ↓
返回 SyncResult
```

### 5.3 启动自动同步流程

```text
应用启动
  ↓
DeviceManager.loadFromCache()
  ↓
检查 miHome.authStoragePath 是否存在
  ↓
有可复用会话？
  ├─ 否：返回缓存并提示扫码登录
  └─ 是：按缓存年龄判断是否触发 syncFromCloud()
```

缓存策略：
- 启动先读缓存，保证界面尽快可见。
- 当存在可复用登录态且缓存超过 10 分钟时，后台自动触发同步。
- 用户手动点击刷新时，始终强制同步。

## 6. 数据存储设计

### 6.1 SQLite 表结构

#### 6.1.1 devices 表

```sql
CREATE TABLE devices (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  model TEXT NOT NULL,
  type TEXT NOT NULL,
  home_id TEXT NOT NULL,
  room_id TEXT,
  room_name TEXT,
  online INTEGER,
  control_capability TEXT NOT NULL,
  control_context TEXT,
  extra TEXT,
  synced_at DATETIME NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_home_id ON devices(home_id);
CREATE INDEX idx_devices_room_id ON devices(room_id);
CREATE INDEX idx_devices_type ON devices(type);
```

#### 6.1.2 rooms 表

```sql
CREATE TABLE rooms (
  id TEXT PRIMARY KEY,
  home_id TEXT NOT NULL,
  name TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_rooms_home_id ON rooms(home_id);
```

### 6.2 缓存策略

| 数据类型 | 策略 | 说明 |
|----------|------|------|
| 设备列表 | 全量替换 | 每次同步后以云端结果为准 |
| 房间列表 | 全量替换 | 与设备同步同批更新 |
| 本地展示信息 | 局部更新 | 用户修改备注/房间时更新 |
| 控制上下文 | JSON 存储 | 保存云控元信息 |

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| SessionMissing | 未登录或无可用认证文件 | 提示用户扫码登录 |
| SessionExpired | 登录态过期 | 触发重新扫码 |
| QrCodeExpired | 二维码过期 | 允许重新生成二维码 |
| SyncFailed | 云端同步失败 | 保留旧缓存并提示重试 |
| CacheWriteFailed | 缓存写入失败 | 记录日志并回退本次更新 |
| MappingFailed | 云端结构映射失败 | 记录错误样本并跳过异常项 |

### 7.2 错误码定义

```typescript
enum DeviceManageErrorCode {
  SUCCESS = 0,
  SESSION_MISSING = 1001,
  SESSION_EXPIRED = 1002,
  QR_CODE_EXPIRED = 1003,
  SYNC_FAILED = 1004,
  CACHE_WRITE_FAILED = 1005,
  MAPPING_FAILED = 1006,
}
```
