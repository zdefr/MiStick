## 2. 模块概述

### 2.1 职责

设备管理模块负责米家设备的发现、添加、信息管理和分组管理，为设备控制模块提供设备数据支持。

### 2.2 功能范围

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 设备发现 | 扫描局域网内的米家设备 | P0 |
| 设备添加 | 添加设备到管理列表 | P0 |
| 设备信息管理 | 设备名称、房间、备注等 | P0 |
| 设备分组 | 按房间/类型分组 | P1 |
| 设备缓存 | 本地缓存设备信息 | P0 |
| 设备删除 | 移除设备 | P1 |

### 3.2 核心类说明

#### 3.2.1 DeviceManager

设备管理模块的主控类，单例模式。

| 属性 | 类型 | 说明 |
|------|------|------|
| devices | Map<string, Device> | 设备缓存 |
| rooms | Map<string, Room> | 房间信息 |
| deviceService | DeviceService | 设备服务实例 |

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| discoverDevices | - | Promise<Device[]> | 发现设备 |
| addDevice | device: Device | Promise<void> | 添加设备 |
| removeDevice | deviceId: string | Promise<void> | 删除设备 |
| updateDevice | device: Device | Promise<void> | 更新设备 |
| getDevice | deviceId: string | Device \| null | 获取设备 |
| getAllDevices | - | Device[] | 获取所有设备 |
| getDevicesByRoom | roomId: string | Device[] | 按房间获取 |
| getDevicesByType | type: DeviceType | Device[] | 按类型获取 |
| addRoom | room: Room | Promise<void> | 添加房间 |
| removeRoom | roomId: string | Promise<void> | 删除房间 |
| getAllRooms | - | Room[] | 获取所有房间 |
| syncFromCloud | - | Promise<void> | 从云端同步 |

#### 3.2.2 Device

设备实体类。

```typescript
interface Device {
  id: string;           // 设备 ID (DID)
  name: string;         // 设备名称
  type: DeviceType;     // 设备类型
  model: string;        // 设备型号
  ip: string;           // IP 地址
  token: string;        // 设备 Token
  roomId?: string;      // 所属房间 ID
  status: DeviceStatus; // 当前状态
  capabilities: string[]; // 支持的能力
  createdAt: Date;      // 创建时间
  updatedAt: Date;      // 更新时间
}
```

#### 3.2.3 DeviceType 枚举

```typescript
enum DeviceType {
  Light = 'light',           // 灯
  Switch = 'switch',         // 开关
  Socket = 'socket',         // 插座
  Curtain = 'curtain',       // 窗帘
  AirConditioner = 'ac',     // 空调
  Sensor = 'sensor',         // 传感器
  Gateway = 'gateway',       // 网关
  Unknown = 'unknown'        // 未知
}
```

#### 3.2.4 DeviceStatus 接口

```typescript
interface DeviceStatus {
  power: boolean;           // 开关状态
  brightness?: number;      // 亮度 (0-100)
  colorTemp?: number;       // 色温
  color?: [number, number, number]; // RGB 颜色
  temperature?: number;     // 温度 (空调)
  humidity?: number;        // 湿度
  battery?: number;         // 电量
  lastUpdate: Date;         // 最后更新时间
}
```

## 4. 接口设计

### 4.1 内部接口 (IPC)

#### 4.1.1 获取设备列表

```typescript
// 渲染进程 → 主进程
ipcRenderer.invoke('device:getAll')

// 主进程处理
ipcMain.handle('device:getAll', async () => {
  return deviceManager.getAllDevices();
});
```

#### 4.1.2 添加设备

```typescript
// 请求参数
interface AddDeviceRequest {
  device: Device;
}

// 响应
interface AddDeviceResponse {
  success: boolean;
  error?: string;
}

// 调用
ipcRenderer.invoke('device:add', { device });
```

#### 4.1.3 更新设备

```typescript
// 请求参数
interface UpdateDeviceRequest {
  deviceId: string;
  updates: Partial<Device>;
}

// 调用
ipcRenderer.invoke('device:update', { deviceId, updates });
```

#### 4.1.4 删除设备

```typescript
// 请求参数
interface RemoveDeviceRequest {
  deviceId: string;
}

// 调用
ipcRenderer.invoke('device:remove', { deviceId });
```

#### 4.1.5 发现设备

```typescript
// 调用
ipcRenderer.invoke('device:discover');

// 返回
interface DiscoverDevicesResponse {
  devices: DiscoveredDevice[];
}

interface DiscoveredDevice {
  did: string;
  model: string;
  ip: string;
  token?: string;
}
```

## 6. 数据存储设计

### 6.1 SQLite 表结构

#### 6.1.1 devices 表

```sql
CREATE TABLE devices (
  id TEXT PRIMARY KEY,              -- 设备 ID (DID)
  name TEXT NOT NULL,               -- 设备名称
  type TEXT NOT NULL,               -- 设备类型
  model TEXT,                       -- 设备型号
  ip TEXT,                          -- IP 地址
  token TEXT,                       -- 设备 Token
  room_id TEXT,                     -- 所属房间 ID
  capabilities TEXT,                -- 能力列表 (JSON)
  status TEXT,                      -- 状态 (JSON)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_room ON devices(room_id);
CREATE INDEX idx_devices_type ON devices(type);
```

#### 6.1.2 rooms 表

```sql
CREATE TABLE rooms (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  device_ids TEXT,                  -- 设备 ID 列表 (JSON)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 缓存策略

| 数据类型 | 缓存策略 | 过期时间 |
|----------|----------|----------|
| 设备列表 | 启动时加载，变更时更新 | 无过期 |
| 设备状态 | 按需加载，定期刷新 | 5 分钟 |
| 房间信息 | 启动时加载 | 无过期 |

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| DeviceNotFound | 设备 ID 不存在 | 返回 null，UI 提示 |
| DeviceOffline | 设备离线 | 标记状态为离线，提示用户 |
| TokenInvalid | Token 无效 | 提示重新获取 Token |
| NetworkError | 网络错误 | 重试 3 次，失败后提示 |

### 7.2 错误码定义

```typescript
enum DeviceErrorCode {
  SUCCESS = 0,
  DEVICE_NOT_FOUND = 1001,
  DEVICE_OFFLINE = 1002,
  TOKEN_INVALID = 1003,
  NETWORK_ERROR = 1004,
  DISCOVER_FAILED = 1005,
  SYNC_FAILED = 1006
}
```
