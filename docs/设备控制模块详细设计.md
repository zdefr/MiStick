## 2. 模块概述

### 2.1 职责

设备控制模块负责对已同步设备执行状态查询和控制指令下发，并将控制结果反馈给 UI 与日志模块。首期正式控制路线仅保留云控。

### 2.2 首期范围

**首期必须实现：**
- 开关类设备的打开/关闭
- 灯类设备的亮度调节
- 支持设备状态查询
- 优先通过 `mijia-api` 云控执行控制
- 云控不可用时，返回明确错误并提示稍后重试
- 对不可控设备返回明确错误

**首期可选实现：**
- 色温调节
- 简单批量控制
- 云控服务健康状态展示

**首期不做：**
- 定时控制
- 自动化场景
- OpenClaw 相关控制

### 3.2 核心类说明

#### 3.2.1 DeviceController

设备控制模块主控类，负责参数校验、路由选择、错误转换和状态回写。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `turnOn` | `deviceId` | `Promise<CommandResult>` | 打开设备 |
| `turnOff` | `deviceId` | `Promise<CommandResult>` | 关闭设备 |
| `setBrightness` | `deviceId, level` | `Promise<CommandResult>` | 设置亮度 |
| `setColorTemp` | `deviceId, value` | `Promise<CommandResult>` | 设置色温 |
| `getStatus` | `deviceId` | `Promise<DeviceStatus>` | 查询设备状态 |
| `batchControl` | `commands` | `Promise<CommandResult[]>` | 批量控制 |

#### 3.2.2 ControlRoutePlanner

控制路由规划器，负责根据设备能力与服务可用性决定控制路径。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `pickRoute` | `device, action` | `ControlRoute` | 决定优先走云控还是本地控制 |
| `canUseCloud` | `device` | `boolean` | 判断是否具备云控条件 |
| `canUseLocal` | `device` | `boolean` | 判断是否具备本地控制条件 |

#### 3.2.3 CloudControlService

云控适配层客户端，封装对 MiHome Bridge Service 的调用。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `control` | `CloudControlRequest` | `Promise<CommandResult>` | 调用桥接服务执行云控命令 |
| `getStatus` | `deviceId, cloudContext` | `Promise<DeviceStatus>` | 查询设备状态 |
| `healthCheck` | - | `Promise<boolean>` | 检查桥接服务可用性 |

#### 3.2.4 LocalControlService

本地控制适配层客户端，封装对 `python-miio` 服务的调用。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `control` | `LocalControlRequest` | `Promise<CommandResult>` | 调用本地控制服务执行命令 |
| `getStatus` | `deviceId, localContext` | `Promise<DeviceStatus>` | 查询设备状态 |
| `healthCheck` | - | `Promise<boolean>` | 检查本地控制服务可用性 |

### 3.3 数据结构

```typescript
interface CloudControlContext {
  did: string;
  props?: Record<string, { siid: number; piid: number }>;
  actions?: Record<string, { siid: number; aiid: number }>;
}

interface LocalControlContext {
  ip?: string;
  token?: string;
  protocol?: string;
}

interface ControlCommand {
  deviceId: string;
  action: 'turnOn' | 'turnOff' | 'setBrightness' | 'setColorTemp';
  payload?: {
    level?: number;
    value?: number;
  };
}

interface ControlRoute {
  type: 'cloud' | 'local' | 'none';
  reason?: string;
}

interface CommandResult {
  success: boolean;
  deviceId: string;
  action: string;
  route?: 'cloud' | 'local';
  errorCode?: ControlErrorCode;
  errorMessage?: string;
  timestamp: string;
}

interface DeviceStatus {
  power?: boolean;
  brightness?: number;
  colorTemp?: number;
  online?: boolean;
  lastUpdate: string;
}
```

### 3.4 路由判断规则

| 场景 | 处理方式 |
|------|----------|
| 设备不存在 | 返回 `DEVICE_NOT_FOUND` |
| 设备存在但无任何控制能力 | 返回 `DEVICE_NOT_CONTROLLABLE` |
| 有云控能力且桥接服务可用 | 优先走 `cloud` |
| 云控失败且本地回退可用 | 走 `local` |
| 仅有本地控制能力 | 直接走 `local` |
| 两条路线都不可用 | 返回 `CONTROL_ROUTE_NOT_FOUND` |

## 5. 流程设计

### 5.1 单设备控制流程

```text
用户点击开关
  ↓
UI 调用 IPC: device:control
  ↓
DeviceController 查询设备信息
  ↓
ControlRoutePlanner 选择控制路径
  ↓
优先调用 CloudControlService.control()
  ↓
失败且具备本地回退条件？
  ├─ 否：直接返回错误
  └─ 是：调用 LocalControlService.control()
  ↓
成功后回写内存态状态
  ↓
返回 CommandResult 给 UI
```

### 5.2 亮度调节流程

```text
用户拖动亮度滑块
  ↓
UI 侧防抖 300ms
  ↓
device:control(action=setBrightness)
  ↓
DeviceController 校验范围 0-100
  ↓
优先调用云控
  ↓
必要时回退到本地控制
```

### 5.3 状态查询流程

```text
UI 打开设备详情
  ↓
device:getStatus
  ↓
优先使用云端状态查询
  ↓
若云端不可用且存在本地上下文，则回退本地查询
  ↓
返回最新状态并更新缓存
```

## 6. 适配层设计

### 6.1 云控适配层要求

- 控制服务接收统一结构的 `deviceId + action + cloudContext`。
- 适配层内部负责把语义动作映射到 `did/siid/piid/aiid`。
- `mijia-api` 的第三方细节不泄露到 Electron 主进程。
- 桥接服务需要支持会话复用和鉴权失效重试。

### 6.2 本地控制适配层要求

- 控制服务只接收统一结构的 `deviceId + action + localContext`。
- 适配层内部可按 `protocol` / `model` 选择不同设备客户端。
- `python-miio` 和设备协议细节不泄露到 Electron 主进程。
- 本地服务不可用时，M02 应向 UI 返回可理解的错误。

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| DeviceNotFound | 设备不存在 | 直接返回错误 |
| DeviceNotControllable | 设备不可控 | UI 显示禁用态 |
| ControlRouteNotFound | 无可用控制路径 | 提示当前设备暂不可控 |
| CloudControlUnavailable | 云控服务不可用 | 尝试本地回退或提示重试 |
| LocalControlUnavailable | 本地服务不可用 | 返回错误并记录日志 |
| TimeoutError | 控制超时 | 按策略重试 |
| InvalidParams | 参数非法 | 直接返回校验错误 |

### 7.2 错误码定义

```typescript
enum ControlErrorCode {
  SUCCESS = 0,
  DEVICE_NOT_FOUND = 2001,
  DEVICE_NOT_CONTROLLABLE = 2002,
  CONTROL_ROUTE_NOT_FOUND = 2003,
  CLOUD_CONTROL_UNAVAILABLE = 2004,
  LOCAL_CONTROL_UNAVAILABLE = 2005,
  TIMEOUT = 2006,
  INVALID_PARAMS = 2007,
}
```

### 7.3 重试机制

- 对云控超时最多重试 1 次。
- 当云控失败且设备具备本地能力时，允许一次本地回退。
- 对参数错误和设备不可控错误不重试。
