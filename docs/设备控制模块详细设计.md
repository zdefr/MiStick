## 2. 模块概述

### 2.1 职责

设备控制模块负责对已同步设备执行状态查询和控制指令下发，并将控制结果反馈给 UI 与日志模块。

### 2.2 首期范围

**首期必须实现：**
- 开关类设备的打开/关闭
- 灯类设备的亮度调节
- 支持设备状态查询
- 对可控设备执行单设备控制
- 对不可控设备返回明确错误

**首期可选实现：**
- 色温调节
- 简单批量控制

**首期不做：**
- 定时控制
- 颜色调节
- 自动化场景
- OpenClaw 相关控制

### 2.3 依赖关系

```
┌─────────────────┐
│  UI 展示模块    │
│    (M03)        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 设备控制模块    │
│    (M02)        │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
┌─────────┐ ┌──────────────────┐
│设备管理 │ │本地控制适配层    │
│(M01)    │ │Python/HTTP       │
└─────────┘ └──────────────────┘
```

说明：
- M02 不负责米家账号登录与云端同步。
- M02 依赖 M01 提供设备元数据和可选的 `controlContext`。
- 若设备缺少控制上下文，M02 应明确返回“当前设备暂不可控”。

### 3.2 核心类说明

#### 3.2.1 DeviceController

设备控制模块主控类，负责参数校验、能力判断、错误转换和状态回写。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `turnOn` | `deviceId` | `Promise<CommandResult>` | 打开设备 |
| `turnOff` | `deviceId` | `Promise<CommandResult>` | 关闭设备 |
| `setBrightness` | `deviceId, level` | `Promise<CommandResult>` | 设置亮度 |
| `setColorTemp` | `deviceId, value` | `Promise<CommandResult>` | 设置色温 |
| `getStatus` | `deviceId` | `Promise<DeviceStatus>` | 查询设备状态 |
| `batchControl` | `commands` | `Promise<CommandResult[]>` | 批量控制 |

#### 3.2.2 DeviceControlService

本地控制适配层客户端，封装对 Python/HTTP 控制服务的调用。

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `control` | `ControlRequest` | `Promise<CommandResult>` | 调用控制服务执行命令 |
| `getStatus` | `deviceId, context` | `Promise<DeviceStatus>` | 查询设备状态 |
| `healthCheck` | - | `Promise<boolean>` | 检查控制服务可用性 |

### 3.3 数据结构

```typescript
interface ControlContext {
  ip?: string;
  token?: string;
  protocol?: string;
}

interface ControlCommand {
  deviceId: string;
  action: 'turnOn' | 'turnOff' | 'setBrightness' | 'setColorTemp';
  payload?: {
    level?: number;
    value?: number;
  };
}

interface ControlRequest {
  deviceId: string;
  action: string;
  payload?: Record<string, unknown>;
  controlContext: ControlContext;
}

interface CommandResult {
  success: boolean;
  deviceId: string;
  action: string;
  errorCode?: ControlErrorCode;
  errorMessage?: string;
  timestamp: string;
}

interface DeviceStatus {
  power?: boolean;
  brightness?: number;
  colorTemp?: number;
  online?: boolean;
  lastUpdate: string;
}
```

### 3.4 能力判断规则

| 场景 | 处理方式 |
|------|----------|
| 设备不存在 | 返回 `DEVICE_NOT_FOUND` |
| 设备存在但 `isControllable = false` | 返回 `DEVICE_NOT_CONTROLLABLE` |
| 设备存在但缺少 `controlContext` | 返回 `CONTROL_CONTEXT_MISSING` |
| 参数越界 | 返回 `INVALID_PARAMS` |
| 控制服务不可用 | 返回 `CONTROL_SERVICE_UNAVAILABLE` |

## 5. 流程设计

### 5.1 单设备控制流程

```
用户点击开关
  ↓
UI 调用 IPC: device:turnOn / device:turnOff
  ↓
DeviceController 查询设备信息
  ↓
校验设备是否存在、是否可控、是否具备 controlContext
  ↓
调用 DeviceControlService.control()
  ↓
成功后回写内存态状态
  ↓
返回 CommandResult 给 UI
```

### 5.2 亮度调节流程

```
用户拖动亮度滑块
  ↓
UI 侧防抖 300ms
  ↓
device:setBrightness
  ↓
DeviceController 校验范围 0-100
  ↓
调用 controlService.control()
  ↓
成功后更新本地状态 brightness
```

### 5.3 状态查询流程

```
UI 打开设备详情
  ↓
device:getStatus
  ↓
DeviceController 校验 controlContext
  ↓
调用 controlService.getStatus()
  ↓
返回最新状态并更新缓存
```

### 5.4 批量控制流程

```
接收 commands[]
  ↓
逐项校验设备能力
  ↓
按设备维度并发执行
  ↓
汇总 CommandResult[]
  ↓
返回聚合结果
```

## 6. 本地控制适配层设计

### 6.1 架构

```
┌─────────────────────────────────────────────────────────┐
│               Local Device Control Service              │
│                    FastAPI + Adapter                    │
├─────────────────────────────────────────────────────────┤
│  /health   /api/control   /api/status                   │
│  Adapter Resolver   Device Client Pool                  │
└─────────────────────────────────────────────────────────┘
```

### 6.2 适配层要求

- 控制服务只接收统一结构的 `ControlRequest`。
- 适配层内部可按 `protocol` / `model` 选择不同设备客户端。
- 第三方库和设备协议细节不泄露到 Electron 主进程。
- 健康检查失败时，M02 应向 UI 返回可理解的错误。

### 6.3 示例代码结构

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/health")
async def health():
    return {"ok": True}

@app.post("/api/control")
async def control_device(payload: dict):
    return {"success": True, "deviceId": payload["deviceId"]}

@app.post("/api/status")
async def get_status(payload: dict):
    return {"power": True, "lastUpdate": "2026-03-21T10:00:00.000Z"}
```

## 7. 异常处理

### 7.1 异常类型

| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| DeviceNotFound | 设备不存在 | 直接返回错误 |
| DeviceNotControllable | 设备不可控 | UI 显示禁用态 |
| ControlContextMissing | 缺少控制上下文 | 提示当前设备暂不可控 |
| ControlServiceUnavailable | 控制服务不可用 | 提示重试并记录日志 |
| TimeoutError | 控制超时 | 按策略重试 |
| InvalidParams | 参数非法 | 直接返回校验错误 |

### 7.2 错误码定义

```typescript
enum ControlErrorCode {
  SUCCESS = 0,
  DEVICE_NOT_FOUND = 2001,
  DEVICE_NOT_CONTROLLABLE = 2002,
  CONTROL_CONTEXT_MISSING = 2003,
  CONTROL_SERVICE_UNAVAILABLE = 2004,
  TIMEOUT = 2005,
  INVALID_PARAMS = 2006,
}
```

### 7.3 重试机制

- 对 `TimeoutError` 最多重试 2 次。
- 对 `ControlServiceUnavailable` 不自动重试，直接提示用户。
- 对参数错误和设备不可控错误不重试。
