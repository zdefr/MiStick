## 3. 系统概述

### 3.1 系统目标

构建一个运行在 Windows 桌面上的轻量级米家设备控制工具，具备：
1. **设备直接控制能力** - 单个设备的开关、调节控制
2. **批处理/场景联动能力** - 通过 OpenClaw AI Agent 执行复杂场景
3. **便利贴形态** - 桌面常驻、简洁直观

### 3.2 用户特征

- 拥有米家智能家居设备的 Windows 用户
- 需要快速控制设备，无需打开完整 App
- 可能有批处理/自动化需求的进阶用户

### 3.3 假设与依赖

| 依赖项 | 说明 |
|--------|------|
| python-miio 库 | 米家设备控制核心库 |
| OpenClaw (可选) | WSL 中运行的 AI Agent 框架 |
| Node.js | Electron 运行环境 |

## 6. 接口设计

### 6.1 内部接口

#### 6.1.1 Electron 主进程 ↔ 渲染进程

| 接口名 | 方向 | 说明 |
|--------|------|------|
| `getDeviceList` | R→M | 获取设备列表 |
| `controlDevice` | R→M | 控制设备 |
| `getBatchScripts` | R→M | 获取批处理脚本列表 |
| `executeBatch` | R→M | 执行批处理脚本 |
| `updateConfig` | R→M | 更新配置 |

#### 6.1.2 Electron ↔ Python 设备服务

| 接口名 | 方法 | 路径 | 说明 |
|--------|------|------|------|
| 设备列表 | GET | `/api/devices` | 获取所有设备 |
| 设备状态 | GET | `/api/devices/:id/status` | 获取设备状态 |
| 设备控制 | POST | `/api/devices/:id/control` | 控制设备 |
| 批量控制 | POST | `/api/batch/control` | 批量控制 |

#### 6.1.3 Electron ↔ OpenClaw 服务

| 接口名 | 方法 | 路径 | 说明 |
|--------|------|------|------|
| 发送指令 | POST | `/api/chat` | 发送自然语言指令 |
| Agent 状态 | GET | `/api/status` | 获取 Agent 状态 |
| 执行历史 | GET | `/api/history` | 获取执行历史 |

### 6.2 外部接口

#### 6.2.1 米家设备接口

使用 `python-miio` 库提供的接口：

```python
from miio import Device

device = Device(ip, token)
device.send("set_power", ["on"])  # 开
device.send("set_power", ["off"]) # 关
```

## 7. 数据设计

### 7.1 核心数据实体

#### 7.1.1 设备 (Device)

```typescript
interface Device {
  id: string;           // 设备 ID (DID)
  name: string;         // 设备名称
  type: DeviceType;     // 设备类型
  model: string;        // 设备型号
  ip: string;           // IP 地址
  token: string;        // 设备 Token
  roomId?: string;      // 所属房间 ID
  status: DeviceStatus; // 当前状态
  capabilities: string[]; // 支持的能力
}
```

#### 7.1.2 房间 (Room)

```typescript
interface Room {
  id: string;
  name: string;
  deviceIds: string[];
}
```

#### 7.1.3 批处理脚本 (BatchScript)

```typescript
interface BatchScript {
  id: string;
  name: string;
  description: string;
  filePath: string;
  registered: boolean;  // 是否已注册到 OpenClaw
  lastExecuted?: Date;
  status: 'success' | 'failed' | 'pending';
}
```

#### 7.1.4 用户配置 (UserConfig)

```typescript
interface UserConfig {
  userId: string;
  miAccount?: string;   // 米家账号
  deviceToken: string;  // 米家登录 Token
  openclawUrl: string;  // OpenClaw 服务地址
  openclawToken?: string;
  windowSettings: WindowSettings;
  theme: 'light' | 'dark';
}
```

### 7.2 数据存储方案

| 数据类型 | 存储方式 | 位置 |
|----------|----------|------|
| 用户配置 | JSON 文件 | `%APPDATA%/mijia-sticky/config.json` |
| 设备缓存 | SQLite | `%APPDATA%/mijia-sticky/devices.db` |
| 日志数据 | SQLite + JSONL 归档 | `%APPDATA%/mijia-sticky/logs.db` + `%APPDATA%/mijia-sticky/logs/archive/` |
| 批处理脚本 | 文件系统 | `C:\mijia-scripts\` |

### 7.3 日志存储架构

采用**冷热数据分离**策略：

```
┌─────────────────────────────────────────────────────────┐
│                      Logger                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────────┐      ┌─────────────────────────┐  │
│  │  热日志 (SQLite) │      │   冷日志 (JSONL 归档)    │  │
│  │                 │      │                         │  │
│  │  - 最近 7 天日志  │      │   - 7 天前日志归档       │  │
│  │  - 支持 SQL 查询  │      │   - 压缩存储            │  │
│  │  - 索引优化     │      │   - 用于审计/导出       │  │
│  └─────────────────┘      └─────────────────────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**SQLite 表结构：**

```sql
CREATE TABLE logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  timestamp DATETIME NOT NULL,
  level TEXT NOT NULL CHECK(level IN ('DEBUG', 'INFO', 'WARN', 'ERROR')),
  module TEXT NOT NULL,
  message TEXT NOT NULL,
  meta TEXT,  -- JSON 字符串
  stack TEXT,
  
  INDEX idx_timestamp (timestamp),
  INDEX idx_level (level),
  INDEX idx_module (module)
);
```

**归档策略：**
- 热数据：最近 7 天，存储在 SQLite 中，支持快速查询
- 冷数据：7 天前，归档到 JSONL 文件，可选压缩
- 自动清理：保留最近 30 天归档

## 8. 非功能性设计

### 8.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 启动时间 | < 3 秒 |
| 设备控制响应 | < 2 秒 |
| 内存占用 | < 200MB |
| CPU 空闲占用 | < 5% |

### 8.2 安全设计

| 风险点 | 应对措施 |
|--------|----------|
| Token 泄露 | 加密存储，使用 Windows DPAPI |
| 通信安全 | localhost 通信，不暴露外网 |
| 脚本执行 | 脚本签名验证，用户确认机制 |

### 8.3 可靠性设计

| 方面 | 设计 |
|------|------|
| 异常处理 | 全局错误捕获，友好错误提示 |
| 数据持久化 | 配置变更即时保存，操作日志记录 |
| 服务降级 | OpenClaw 不可用时，基础控制功能正常 |

### 8.4 可扩展性设计

- 模块化设计，各模块低耦合
- 插件化批处理脚本支持
- 配置驱动的设备类型扩展
